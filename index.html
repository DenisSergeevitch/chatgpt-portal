Local browser bridge

Expose authenticated pages to ChatGPT without exposing credentials

Run a local read-and-navigate bridge against a dedicated Chrome profile, then publish only the sanitized portal through a temporary Cloudflare Tunnel.

Start

Run the local bridge

  1. Install dependencies once.

    Run this from the local clone of this repository.

    cd chatgpt-portal && npm install
  2. Start share mode with an allowlist.
    CHATGPT_PORTAL_ALLOWLIST=https://intranet.example.com npm run share

    Replace the example origin with the private site origin or path you want this session to inspect.

  3. Log in inside the dedicated Chrome window.

    The bridge opens or attaches to a Chrome profile under local ignored state. Use that browser to authenticate normally.

  4. Wait for the CLI to print the public URL.
    Share this URL with ChatGPT: https://random-name.trycloudflare.com/s/SESSION_TOKEN/view

Share

Give ChatGPT the tokenized view URL

Share mode prints the final public URL directly:

Share this URL with ChatGPT:
https://random-name.trycloudflare.com/s/SESSION_TOKEN/view

Share only the full URL printed after Share this URL with ChatGPT:. Do not share cookies, browser profile files, local databases, or the intranet URL directly.

Routes

What ChatGPT can use

/view
Read the current sanitized page snapshot.
/page?url=...
Open a page and return its sanitized snapshot.
/open?url=...
Navigate the dedicated browser tab to an allowlisted URL.
/links?url=...
List navigable links from a page.
/crawl?scope=...&limit=...
Crawl allowlisted same-origin pages into the local search index.
/search?q=...
Search locally indexed sanitized snapshots.
/click?id=...
Click only navigation-like controls from the latest snapshot.

Safety

What the bridge blocks

  • No raw reverse proxy into the private site.
  • No cookies, bearer tokens, auth headers, localStorage, sessionStorage, hidden inputs, CSRF fields, password fields, scripts, or raw forms in snapshots.
  • No non-allowlisted origins or paths.
  • No form submissions, uploads, downloads, saves, sends, deletes, approvals, charges, refunds, resets, publishes, or invites.
  • No long-lived public hostname in v1; use temporary `trycloudflare.com` tunnel URLs.

Stop

Revoke access

Stop `cloudflared` to kill the public tunnel. Stop the Node process to invalidate the local token. You can also call shutdown locally with the printed token:

curl -X POST "http://127.0.0.1:7777/shutdown?token=SESSION_TOKEN"
